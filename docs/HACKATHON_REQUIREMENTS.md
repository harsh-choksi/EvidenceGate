## Verified project requirements

- Build a working project with **Codex and GPT-5.6**. Their use must be meaningful, not incidental or decorative.
- Enter one category. EvidenceGate's selected category is **Developer Tools**, which explicitly includes testing, DevOps, agentic workflows, and security.
- The project must install and run consistently on its stated platform and function as depicted in the video and description.
- A pre-existing project must have been meaningfully extended during the submission period. Prior and new work must be distinguished with dated evidence such as Codex logs or commit history.
- The entrant must be authorized to use every third-party SDK, API, dataset, trademark, and other submitted material, and must comply with applicable licenses.
- Entrant and team eligibility must be confirmed directly against Section 3 of the Official Rules. This repository does not assert anyone's eligibility.

The Responses API and OpenAI web search are central EvidenceGate product decisions. They are not listed here as separate hackathon-wide requirements.

## Verified submission requirements

Nothing in this section is marked complete merely because a placeholder or plan exists.

- [ ] Developer Tools category selected in the submission form.

- [ ] All submission materials are in English, or the required English translations are supplied for the video, description, testing instructions, and other materials.
- [ ] Any pre-existing code and all third-party work are disclosed as applicable.

## Judging alignment

Eligible submissions first face a viability pass/fail review for theme fit and use of required APIs/SDKs. Stage-two submissions are evaluated with equally weighted criteria:

- **Technological Implementation:** thorough, skillful Codex use and a working, non-trivial implementation.
- **Design:** a complete, coherent, runnable product experience rather than a disconnected proof of concept.
- **Potential Impact:** a credible problem, audience, and demonstrated solution.
- **Quality of the Idea:** creativity, novelty, and differentiation.

The Fail-to-Pass demo, dual-evidence boundary, deterministic policy, and inspectable report should make these qualities visible rather than merely claim them.
